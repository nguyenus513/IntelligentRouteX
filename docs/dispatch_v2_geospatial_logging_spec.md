# Dispatch V2 Geospatial Logging Spec

- Every candidate-level Bronze row must include:
  - `rowType`
  - `entityType`
  - `entityId`
  - `candidateId`
- Candidate id normalization:
  - bundle: `bundle:<bundleId>`
  - anchor: `anchor:<bundleId>:<anchorOrderId>`
  - driver: `driver:<bundleId>:<driverId>`
  - proposal: `proposal:<proposalId>`
  - assignment: `assignment:<assignmentId>`
- Required geospatial fields when available:
  - driver/pickup/dropoff coordinates
  - bundle centroid coordinates
  - pickup/drop cluster radii
  - route spread
  - bundle compactness
  - driver-to-first-pickup distance and ETA
  - stop order and route vector summaries
  - tile summaries and corridor summary
- Provenance is mandatory for traffic/weather/tile summaries:
  - measured
  - estimated
  - fallback
  - missing
- Raw tile images are prohibited. Only compressed summaries and vectors are allowed.
